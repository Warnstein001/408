# 运算器
![[Pasted image 20260311140510.png]]

|   ALU   |  +   |  -   |  $\times$  |  $\div$   |
| :-----: | :--: | :--: | :--------: | :-------: |
| **ACC** | 被加数和 | 被减数差 |    乘积高位    | 被除数<br>余数 |
| **MQ**  |      | <br> | 乘数<br>乘积低位 |     商     |
|  **X**  |  加数  |  减数  |    被乘数     |    除数     |
```ad-info
M 表示主存储器中某个存储单元的地址
(M) 表示地址为M的存储单元中的内容

ACC 表示累加器
(ACC) 表示累加器中的内容

MQ 表示乘商寄存器
(MQ) 表示乘商寄存器中的内容

X 表示操作数寄存器
(X) 表示操作数寄存器中的内容
```
> <mark style="background:rgba(92, 92, 92, 0.2)">假设累加器ACC中已存有前一时刻的运算结果，并作为下述运算中的一个操作数</mark>

## 加法操作过程
1. (M) --> X——取出存放在主存中地址为M的存储单元中的内容(M)（加数），送到操作数寄存器X中。
2. (ACC) + (X) --> ACC——将累加器ACC中的内容(ACC)（被加数）与操作数寄存器中的内容(X)（加数）相加，结果（和）保留在累加器ACC中。   
以上简化描述，强调来源和去向。实际流程如下：
```text
ACC ──→ ALU ──→ ACC
        ↑
X ──────┘
```
## 减法操作过程
1. (M) --> X —— 取出存放在主存中地址为M的存储单元中的内容(M)（加数），送到操作数寄存器X中。
2. (ACC) - (X) --> ACC —— 将累加器ACC中的内容(ACC)（被减数）与操作数寄存器中的内容(X)（减数）相加，结果（差）保留在累加器ACC中。 
## 乘法操作过程
1. (M) --> MQ —— 取出存放在主存中地址为M的存储单元中的内容(M)（乘数），送到乘商寄存器MQ中。
2. (ACC) --> X ——将累加器ACC中的内容(ACC)（被乘数），送到操作数寄存器中。
3. (X) $\times$ (MQ) --> ACC // MQ —— 将操作数寄存器X中的内容(X)（被乘数）与乘商寄存器中的内容(MQ)（乘数）相乘，结果（积）的高位保留在累加器ACC中，低位保留在乘商寄存器MQ中。
```ad-thinking
如何划分高位和低位？
在乘法运算的结果中，高位和低位是按二进制（或十进制）数的位权来划分的：

假设被乘数和乘数都是 n 位（比如 n=16 位），那么它们的乘积最多是 2n 位。

- 乘积的高 n 位：是结果中 “位权更高” 的那一半（比如 16 位乘法结果的第 17~32 位），存到累加器 ACC；
- 乘积的低 n 位：是结果中 “位权更低” 的那一半（比如 16 位乘法结果的第 1~16 位），存到乘商寄存器 MQ。

举个十进制的例子（更直观）：

如果被乘数是123（3 位）、乘数是45（2 位），乘积是5535（4 位）。

- 若按 “结果长度 = 被乘数位数 + 乘数位数”（3+2=5 位，补前导零为05535），则：  
    高位是前 3 位055，低位是后 2 位35。

简单说：高位是乘积中 “位数靠前、数值更大” 的部分，低位是 “位数靠后、数值更小” 的部分，具体划分由寄存器的位数（被乘数 / 乘数的位数）决定。
```
## 除法操作过程
1. (M) --> —— 取出存放在主存中的地址为M的存储单元中的内容(M)（除数），送到操作数寄存器X中。
2. (ACC) $\div$ (X) -->MQ　—— 将累加器ACC中的内容(ACC)（被除数）除以操作数寄存器X中的内容(X)（除数），结果（商）保留在乘商寄存器MQ中，余数保留在累加器

# 存储器
![[Pasted image 20260311153044.png]]
- **存储体**由很多个**存储单元**组成
	- 每个**存储单元**由若干个**存储元件**组成
		- 每个存储元件能存放一位二进制数“0” or “1”
	- 一个存储单元中可存储一串二进制信息，称这串二进制信息为一个**存储字**，这串**二进制信息的位数**称为**存储字长**。（可以是8位，16位，32位等）
- 给每个存储单元都赋予一个编号，称为**存储单元的地址**。
- 存储器**地址寄存器MAR**，用来**存放欲访问的存储单元的地址**。
	- **MAR的位数**（长度），**决定了存储单元的数量**。
```ad-question
上图中的存储体总容量？
16 位 $\times$ $2^{16}$ 个 = $2^{20}$ 位 = $2^{20}$ 比特 = $2^{20}$ b
$2^{20}$ b $\div$ 8 = $2^{17}$ B = 128 KB
```
- 存储器**数据寄存器MDR**，用来**存放从存储体的某个存储单元取出的信息或者准备往某个存储单元存入的信息**。
	- **MDR的位数**（长度），**与存储字长相等**。
```ad-note
主存（内存）的这种按存储单元的地址来实现对其进行写入和读取操作，需要在CPU中的控制器的控制下进行。
```
# 控制器
![[Pasted image 20260311155554.png]]
- 控制器是计算机的神经中枢，由他指挥各个部件自动、协调地工作。
	1. 控制从主存中读取一条指令，称为**取值**过程（阶段）
	2. 对指令进行分析，指出该指令要完成何种操作，并按寻址特征指明操作数的地址，称为**分析**过程（阶段）。
	3. 根据指令的操作码和操作数所在的地址完成某种操作，称为**执行**过程（阶段）。
![[Pasted image 20260311161226.png]]
- 程序计数器**PC用来存放当前欲执行的指令地址**
	-  **PC与MAR之间有一条直接通路**。
	- PC自动形成下一条指令地址（“自动＋１“的功能）。
![[Pasted image 20260311161302.png]]
- 指令寄存器**IR用来存放当前的指令**
	- **IR的内容来自MDR**。
	- **IR中的操作码**（用OP(IR)表示）会送至CU（用**OP(IR)** --> **CU** 表示），用来**分析指令**。
	- **IR中的地址码**（用Ad(IR)表示）作为操作数的地址送至MAR（用**Ad(IR) --> MAR** 表示），用来**从内存中取操作数**。
![[Pasted image 20260311161700.png]]
- 控制单元**CU用来分析当前指令所需完成的操作，并发出各种微操作的命令序列**，用以控制所有被控对象。

# 机器指令简介
![[Pasted image 20260311163418.png]]
- 操作码表示及其所执行的各种操作，例如：取数，存数，加、减、乘、除等
- 地址码表示参加运算的数在内存中的位置。
```ad-question
这些地址码都是规定好的吗？
## 关键点说明：

### 1. **操作码是硬件设计时规定的**

- 这些二进制代码（000001、000010等）在**CPU设计阶段**就已经确定
- CPU内部的**控制单元(CU)**能够识别这些特定的二进制模式
- 比如看到 `000001` 就知道是"取数"操作

### 2. **指令集架构(ISA)**

这个表格展示的是一种**简单模型的指令集**，包括：

- **取数** (Load)：从内存取数据到ACC
- **存数** (Store)：将ACC数据存入内存
- **加** (Add)：执行加法运算
- **乘** (Multiply)：执行乘法运算
- **停机** (Halt)：程序结束

### 3. **为什么是这些编码？**

- **6位操作码** → 最多可表示 2^6 = 64 种不同指令
- 这里只用了5种，剩下的编码可以留给其他指令
- 实际计算机（如x86、ARM）有几百条指令，编码更复杂

### 4. **程序员/编译器必须遵守**

- 编写程序时，必须使用这些规定的操作码
- 高级语言编译后，会转换成这些机器指令

**总结**：这是计算机的"语言字典"，CPU只认识这些规定好的代码！
```
