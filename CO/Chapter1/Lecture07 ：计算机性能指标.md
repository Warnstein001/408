# 什么是计算机性能指标
- 计算机性能指标是用来衡量计算机系统在各种操作和任务中的**工作效率**、**执行速度**以及**处理能力**的标准。
- 计算机系统的性能指标可以帮助**人们评估计算机系统**的整体表现，从而更好地了解系统**在不同方面的优势**。
# 软硬件与计算机性能的关系
## 硬件与计算机系统的关系
- 硬件是构建计算机系统的**物理组件**（eg. CPU、内存、外部设备等）。
- 硬件对于计算机系统的性能有**重要影响**，因为她决定里系统的**计算能力**、**数据传输速率**和**存储容量**。
	- CPU的时钟频率决定了CPU每秒钟可以执行的指令数量。
	- 内存带宽会影响数据的读写速率。
## 软件与计算机系统的关系
- 软件你包括用于控制、管理、应用计算机系统的各类**系统软件**和**应用软件**。
- 软件的优化可以**显著影响**计算机系统的性能，因为**合理的软件算法**和**代码实现**可以有效地利用**计算机硬件资源**。
	- 操作系统的调度算法会影响多任务处理的效率，从而影响系统的响应时间。
	- 在图像处理任务中，优化的软件算法可以减轻CPU和内存的负担，提高图形处理速度。
	- 软件层面的并行计算可以更好的利用多核处理器，提高吞吐量。
```ad-summary
在计算机系统中，软硬件之间的关系密切。
- 优化硬件可以提供更强大的计算和传输能力。
- 优化软件可以更有效的利用这些硬件资源。

他们俩共同的实现更好的系统性能。正确的硬件选择可以为软件提供更好的执行环境，反之亦然。
```
# 计算机硬件的相关性能指标
## 基本性能指标
### 机器字长
- 机器字长指**CPU一次能处理的二进制数据的位数**，也就是构成二进制数据的比特（bit，简写为b）数量。
	- 机器字长与CPU内部用于**整数运算的ALU位数**以及**通用寄存器的宽度**相等。
- 机器字长对计算机系统性能的主要影响如下：
	- 字长越长，**数的表示范围**越大，**精度**越高。
	- 字长越长，**计算精度**越高。
	- 字长还会影响**计算速度**。
- 随着硬件技术的发展，计算机的字长已经从早期的8位逐渐增加到16位、32位、64位甚至更高。
### 主存容量
- 主存容量是指**主存储器（内存）能够存储的最大信息量**。
````ad-exercise
某计算机主存地址为 16 位，每个存储单元的存储字长为 16 位。请回答以下问题：

1. 该主存共有多少个存储单元？
2. 计算该主存的总容量（分别用**位 (bit)**、**字节 (Byte)**、**千字节 (KB)** 表示）。

```ad-hint
- 存储单元数 M=$2^{地址位数}$
- 总容量（位）= 存储单元数 M× 字长 N
```
解：
- **存储单元数量**：
    
    地址为 16 位，可寻址单元数为 216=65536 个。
    
- **总容量计算**：
    
    1. 用 ** 位 (bit)** 表示：
        容量=65536×16=1048576 bit
    2. 用 ** 字节 (Byte)** 表示：
        1048576 bit÷8=131072 B
    3. 用 ** 千字节 (KB)** 表示：
        131072 B÷210=128 KB
````
- **增加主存（内存）容量可以减少程序运行期间对辅存（外存）的访问**，由于**访问内存的速度远大于访问外存的速度**，因此可以提高程序的执行速度，进而提高计算机系统的性能。

### 吞吐量
- 吞吐量是指计算机系统在**单位时间内能够处理的信息量**。
- 影响吞吐量的主要因素：
	- CPU的处理能力
	- 内存（主存）的访问速度
	- 外存（辅存，例如硬盘）的访问速度
### 响应时间
- 响应时间是指从向计算机系统**提交作业开始**，到系统**完成作业为止所需要的时间**。
#### CPU时间
- **CPU执行之间**
	- 执行用户程序本身所花费的时间
> 本质：执行业务逻辑、循环、计算、函数调用等所花费的时间。
- **系统CPU时间**
	- 为执行程序而花费在操作系统上的时间
> 本质：内核为进程提供服务所消耗的 CPU 时间。
```ad-note
很难精确区分一个程序执行过程中的CPU执行时间和系统CPU时间，在没有特别说明的情况下，基于CPU执行时间进行计算机性能评价。
```
#### 其他时间
- 用户访问内存、外存、其他外部设备所花费的时间
## 与运算速度相关的性能指标
### CPU时钟频率和时钟周期
- **CPU时钟信号**是一个**基本定时信号**，它是一种==固定频率==的**脉冲信号**，用于驱动计算机内部各个组件协调工作。
	- 这种脉冲信号的频率被称为**CPU时钟频率**（Clock Rate），基本单位为赫兹（Hz）
	- **CPU时钟周期**（Clock Cycle OR Clock Tick，也可简称clock OR tick）是**CPU时钟频率的倒数**，基本单位为秒（s）。
````ad-thinking
什么是基本定时信号？
**基本定时信号**是计算机中用于**同步和控制**各个部件工作的时钟脉冲信号。

## 通俗理解

就像乐队的**指挥**打拍子：

- 指挥每打一下拍子，乐手就演奏一个音符
- CPU每来一个时钟脉冲，就执行一个微操作
## 核心作用

### 1. **同步作用**

- 确保CPU内部各部件（运算器、控制器、寄存器）**协调工作**
- 所有操作都按照统一的节拍进行

### 2. **时序控制**

- 规定每个操作在**什么时间**进行
- 例如：取指→分析→执行，每个阶段需要几个时钟周期

## 具体示例
```

时钟信号：┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐
         │ │ │ │ │ │ │ │ │ │
         └─┘ └─┘ └─┘ └─┘ └─┘
周期：     T0  T1  T2  T3  T4

操作：    取指  译码  取数  运算  存结果
```

每个**时钟周期**（T0、T1、T2...）就是一个基本定时信号，告诉CPU："现在该做下一步操作了！"

## 总结

**基本定时信号 = 计算机的"心跳" + "节拍器"**，确保所有操作有序、同步地进行！
````

### CPI（Cycle Per Instruction）
- CPI指**执行一条指令所需要的时钟周期数量**。
- 由于不同指令的功能不同，即使功能相同的指令还可能有不同的寻址方式，因此**每条指令执行所需的时钟周期的数量也不同。**
	- 针对一条指令、一类指令、一个程序，他们各自的CPI的含义：
$$
CPI=程序执行所需时钟周期数量 \div 程序所包含的指令数量
$$
		- **一条指令**的CPI：该指令执行所需要的时钟周期数量
> 如果程序仅包含一条指令，则CPI就是这条指令的CPI。

		- **一类指令**（例如算术运算类指令）的CPI：构成该类指令的所有指令执行所需要的时钟周期数量的平均值。
> 若程序仅包含一类指令，则CPI就是这类指令的CPI

		- **一个程序**的CPI：构成该程序的所有指令执行所需要时钟周期数量的平均值。
#### 计算程序的CPI
如果知道某个程序中共有<u>n类不同类型的指令</u>、<u>每类指令的CPI</u>（用 $CPI_i$ 表示）、每类指令的数量在程序所包含指令数量中<u>所占的比例</u>（用 $P_i$ 表示），则该程序的CPI可用下式计算：
$$
CPI = \sum_{i=1}^{n}(CPI_i \times P_i)
$$
```ad-exercise
某个程序包含A、B、C和D共4类指令，它们各自在该程序中所占比例依次为60%、18%、12%、10%，A类指令的CPI为1，B类指令的CPI为2，C类指令的CPl为4，D类指令的CPI为8，则该程序的CPI是多少?
$$
CPI=\sum_{i=1}^{n}(CPI_i \times P_i)=1 \times 60\% + 2 \times 18\% + 4 \times 12\% + 8 \times 10\% = 2.24

$$
```
